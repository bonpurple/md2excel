# Release Notes - v2.1.2 (2026-03-11)

今回の更新では、**改行をまたぐ強調記法の書式継続**に関する不具合を修正しました。  
Markdown 内で太字・斜体の記法が複数行にまたがる場合でも、Excel 出力時に意図した装飾が維持されるようになっています。

---

## 主な修正内容

これまで、次のような Markdown を変換した場合、改行をまたいだ強調記法が正しく解釈されず、各行が通常フォントで出力されることがありました。

```md
## title

**a  
b**
```

修正前は、たとえば以下のように出力される場合がありました。

* A3: `**a`
* A4: `b**`

または、`a` / `b` 自体は分割されても、どちらも通常フォントとして出力されるケースがありました。

v2.1.2 ではこの挙動を修正し、以下のように出力されるよう改善しています。

* A3: `a`（太字）
* A4: `b`（太字）

---

## 対応した記法

今回の修正では、改行をまたぐ以下の強調記法に対応しています。

* `**bold**`
* `__bold__`
* `*italic*`
* `_italic_`

また、次のような改行ケースでも、強調状態が適切に次行へ引き継がれるようになりました。

* 行末の `<br>`
* Markdown のハード改行（行末スペース 2 個以上）
* バックスラッシュによる改行

---

## この修正の意図

md2excel は、設計書や仕様書の Markdown をできるだけ自然な読みやすさで Excel に落とし込むことを目的としています。
そのため、単一行の装飾だけでなく、**文書として自然に書かれた複数行の強調表現**も正しく反映できることを重視しています。

v2.1.2 では、改行を含む Markdown においても、セルの縦展開とインライン装飾の継続がより自然になるよう改善しました。
