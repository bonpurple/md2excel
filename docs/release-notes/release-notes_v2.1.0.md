# Release Notes - v2.1.0 (2026-03-03)

md2excel v2.1.0 は、Markdown の **強調（`*` / `_`）解釈精度**を改善し、CommonMark に近い delimiter run 解決へ移行したリリースです。  
設計書用途で頻出する「複合強調」や「改行（`<br>`）をまたぐ装飾継続」の安定性を高めています。

---

## ハイライト

- `*` / `_` の強調判定を、逐次トグル方式から **delimiter run 解決**へ刷新
- `***` / `___` などの **複合 run（太字＋斜体）**に対応
- `<br>` 分割（縦展開）時の **強調の閉じ／再オープン**を明確化し、装飾崩れを抑止

---

## 変更内容

### Changed（仕様変更・改善）
- `*` / `_` の強調判定を CommonMark 寄りの delimiter run 解決に変更
- delimiter run の境界判定に Unicode の whitespace / punctuation を考慮
- `_` に対する追加条件（誤検出抑止のための条件）を反映
- delimiter run の **rule of 3** を反映
- `***` / `___` のような複合 run（太字＋斜体）や混在 run に対応

### Fixed（不具合修正）
- `<br>` をまたぐ縦展開時、**開いている強調のみを一旦閉じ**、次行で再オープンするように修正  
  （改行を挟んだときに意図せず装飾が崩れるケースを抑止）

---

## 互換性・影響

- Markdown の強調解釈がより CommonMark に寄るため、**従来“たまたま”強調になっていた文字列が強調されなくなる**、またはその逆が起こり得ます。  
  特に `_` を含む単語（識別子・ファイル名等）が多い文書では、出力差分を確認してください。
